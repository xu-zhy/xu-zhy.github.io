  36. 空值处理
    a. NULL 与所有的比较运算符不匹配，所以例如 score < 60 的筛选条件不会把 null 选进去。
    b. 按顺序排序时 null 被处理为最小值（SQLserver）
    c. 集合函数中，除了COUNT(*)计算元组时要把取空值的项计算进去，其他的集合函数都忽略了取空值的项。
    d. 对于空集，集合函数除了 count(*) 给0外，其它给 NULL
    e. <=all(空集）在 SQLserver 为真，因为相当于不需要比了。
    f. 连接时两个 null 不会相连
    g. b>=all(A) 中如果 A 集合存在 NULL，且b 存在空集，即出现两个空集相比较，结果为空集。

  48. 设置事务隔离等级

    read uncommitted: 允许读到其它未提交的脏数据
    read committed：保证不读到脏数据，即其它事务未提交数据；但是允许其他事务在两次读取的间隙修改资源（已提交），故两次读取数据不同
    repeatable read: 在一个事务两次读取一个数据项期间，其他事务不得更新（实际上是不允许删除）该数据（但可以插入新数据），保证了两次读取间的数据相同，保证可重复读；但是可能因此幻象读，即读到其它事务插入的新数据。
    serializable: 防止数据在事务执行间更新或插入数据。【最高等级，避免幻象读】

  49. 事务锁
    在 repeatable read 事务等级下，如果事务对某行数据进行了修改，则会获得排他锁，该锁在事务结束前一直拥有，不允许其它事务对该行进行读取和更新。
    当两个持有同一数据的共享锁的事务都希望升级为排他锁时，排他锁要求其它事务不得持有该数据的任何锁，故两个事务都会等待对方放弃共享锁，故陷入死锁。
