渐进分析与算法复杂度分析

  • Big-O：$f(n) = O(g(n))$
    • $f(n)$ 的增长速度不大于$g(n)$：$\lim_{n \to \infty}\frac{f(n)}{g(n)} = c, c \in[0, \infty)$
  • Little-O: $f(n) = o(g(n))$
    • $f(n)$ 的增长速度小于$g(n)$：$\lim_{n \to \infty}\frac{f(n)}{g(n)} = 0$
  • Big-Theta：$f(n) = \Theta(g(n))$
    • $f(n)$ 的增长速度等同于$g(n)$：$\lim_{n \to \infty}\frac{f(n)}{g(n)} = c, c \in(0, \infty)$
  • Big-Omega：$f(n) = \Omega(g(n))$
    • $f(n)$ 的增长速度不小于$g(n)$：$\lim_{n \to \infty}\frac{f(n)}{g(n)} = c, c \in(0, \infty]$
  • Little-Omega：$f(n) = \omega(g(n))$
    • $f(n)$ 的增长速度大于$g(n)$：$\lim_{n \to \infty}\frac{f(n)}{g(n)} = \infty$
  • 递归分析：主定理 Master Theorem

树

  • 基本概念
    • 树：无重复边和自环，分为内部节点和叶子节点。
    • 树的节点的度、树的度、树的节点的深度、树的深度
    • m 元树、正则 m 元树（full m-ary tree）、二叉树（二元有根有序树）
    • 平衡树：叶子节点深度为树高度或树高度减一（不严谨）
  • 树的定理
    • n 个节点有 n-1 条边
    • 正则 m 元树有 k 个内部节点，则有 mk+1 个节点
    • 对于一个正则 m 元树，n 个节点，k 个内部节点，l 个叶子，有
      • k=(n−1)/m,l=[(m−1)n+1]/m
      • n=mk+1,l=(m−1)k+1
      • n=(ml−1)/(m−1), k=(l−1)/(m−1)
    • 高度为 h 的 m 元树最多有 $m^h$ 叶子
    • 如果一个高度为 h 的 m 元树有 l 叶子，则 h >= $log_m l$
  • Parental Tree（结点指向父亲），采用数组即可实现
  • 二叉树
    • 概念
      • **正则二叉树(full binary trees)**：内部节点度数都为2
      • **完美二叉树/满二叉树(perfect binary trees)**：所有叶子结点同高度，所有内部节点都有两个孩子结点
        • 高度 h 的满二叉树有 $2^{h+1}-1$ 节点
        • 高度 h 的满二叉树有 $2^h$ 叶子节点
      • **完全二叉树(complete binary trees)**：从左到右填满对应高度的完美二叉树，最后一层允许不满
        • n 个节点的完全二叉树的高度为 h = $\lfloor log_2 n \rfloor$
        • 完全二叉树适合使用数组存储，首位不使用，存储顺序与广搜一致。结点k的儿子结点为 2k(k<<1) 和 2k+1 (左儿子结点|1)，父亲结点为 k/2(k>>1);
      • 平衡二叉树：Height balancing(AVL)，Null-path-length balancing(Red-black trees)，Weight balancing(Boundedbalance trees)
    • 遍历：前序、中序、后序遍历
    • 实现：
    • LCRS：左儿子-右兄弟树。将一棵普通树转化为 LCRS 的算法称为 Knuth Transform
    • Huffman Tree
      • 哈夫曼算法：算法维护一个非递减的权值序列，每一趟从序列中取出最靠前的两个权值，构造他们的父亲结点；新结点的权值是这两个权值的和。从序列中删去两个旧权值，并向序列增加这个新权值。持续这个迭代过程，直到序列只剩下一个元素时结束。
      • 哈夫曼树是正则二叉树，一棵有 n 个叶子节点的哈夫曼树共有 2n-1 个节点（归纳证明）
      • 实现：
      • Huffman 编码也是一种信源压缩编码，当确知信源符号的概率分布时，使用 Huffman 编码理论上达到了压缩的极限
    • Binary Search Tree(BST)
      • 结点的值大于左子树所有的值，小于右子树所有的值，可以通过中序遍历得到递增数组。
      • 搜索、插入的复杂度$O(h)$
      • 删除：
        • 如果删除的是叶子节点，直接删除
        • 如果删除的是只有一个子节点的节点，将子节点上提即可
        • 如果删除的是同时有左右子节点的节点，取右子树的最小值节点（右子树最左边的节点）取代待删除节点的位置

          • 如果右子树最小值节点有右子树，还需要将右子树上提到右子树最小节点的位置（实际上不需要去判定）
    • AVL Tree
  • 优先队列

图

  • 基本概念：
    • 对于有向图，边数=顶点出度之和=顶点入度之和；对于无向图，边数×2=顶点度数之和
    • 对于非空且非平凡的图，至少有一对顶点具有相同的度。
      • 推论：度为奇数的顶点的个数为偶数。
    • 完全图、正则图、子图
    • Walk/Chain、Trail/Tour、Path -> 欧拉图、哈密尔顿图
    • 距离、直径
  • 图的存储
    • 邻接矩阵
    • 关联矩阵（性质）
    • 链式前向星 ：类似于稀疏矩阵的存储方式（CSR）。
    • 邻接表
  • 图的遍历
    • 广度优先（BFS）：（使用队列）
      • 从图中找出一个未被标记的顶点，标记为已访问（marked），然后入队
      • 如果队不空，首元素出队，遍历该顶点的邻接顶点，如果邻接顶点已访问则通过，如果未访问则标记为已访问并入队。
      • 如果存在未访问顶点，重复步骤1，否则结束BFS；
    • 深度优先（DFS）：（使用栈）
      • 从图中找一个未被访问的顶点，标记为已访问(marked)，设为当前顶点（入栈）；
      • 从当前顶点开始（栈顶顶点），如果其存在邻接顶点未被访问，则将其入栈并标记为已访问，否则回溯，直至栈顶顶点存在未访问的邻接顶点。
      • 直到没有顶点未被访问（栈空状态下不一定所有顶点都被访问），结束DFS
  • 图的算法

    • 连通子图：BFS需要首先找一个顶点入队，使用并查集记录其后入队的顶点，若队列为空，继续找一个未被访问的顶点入队，同时新建一个并查集记录其后入队的顶点，这样就能将原来的图分为几个连通分支。每一个并查集就是一个连通子图。

    • 二部图：

      • 从图中选择一个顶点，标记为V1,入队；
      • 如果队列非空，队首顶点出队，如果该顶点标记为V1,那么它的所有未被访问的邻接顶点标记为V2,且其已经被访问的邻接顶点不能是标记为V1，如果是则原图不是二部图。
      • 直至队空，此时图为二部图。

    • 拓扑排序：

      • 有向无环图（directed acyclic graph ,DAG）+自环构成一个偏序关系，也就是说一个DAG表示的拟序关系。
        • 定理：在一个DAG里没有长度超过1的环。（允许长度为1的自环）
      • 拓扑排序定义：给定一个图G={V,E},|V|=n,图上的拓扑排序就是图的顶点的一种排序，使得如果存在一条有向路径从u到v（意味着u≼v），则u必须在序列中位于v之前。注意，如果u和v之间没有路径，则u和v在图中的顺序没有要求。因此拓扑排序不是唯一的。拓扑排序实际上也是一种拟序关系的表示
      • 拓扑排序的存在性证明
        • 引理：一个DAG中一定存在至少一个零入度顶点和一个零出度顶点。（证明：利用上面DAG没有环的定理和鸽笼原理证明）
        • 引理：DAG的子图一定是DAG.因此DAG的子图仍然至少有一个零入度顶点。
        • 定理：一个图是DAG当且仅当它具有拓扑排序。
      • 实现：需要一个表记录各顶点入度，一个队列（数组即可）记录当前的零入度顶点。 在拥有这两个数据结构的情况下，将队首顶点出队，其邻接顶点的入度减一，如果该顶点的入度为0，则入队。进行n次出队（顶点数）即可得到拓扑排序（保存在队列数组里）
      • 应用：判断环（当循环走到某一步时，队列中没有顶点了，即此时子图中没有零入度顶点，那么意味着原图存在至少一个环。也就是说判别图是否有环的复杂度为$Θ(n+m)$）

    • 最小生成树

      • 性质
      • Prim 算法：$O(n^2)$，复杂度对图的稀疏稠密没要求
        • 维护一个包含（Visited, Distance, Parent）三元组的表
        • 选择一个根顶点设置其distance=0,其他顶点的distance=∞,所有顶点的visited设置为False,所有顶点的parent为空。接下来循环（Θ(n)）：选择距离最小的unvisited顶点(Θ(|V|-|S|))，标记其为visited，然后对于该顶点的每一个unvisited邻接顶点，如果其distance大于该边的权值，则更新为该边权值并把parent设为该顶点。循环直至没有带有有限距离的未访问顶点（存在无限距离未访问顶点是非连通图）。
        • 小顶堆优化：每一次选择最小距离的顶点只需pop后判断是否已经访问，若已经访问再次pop，可见为O(lnn)，共有O(n)次，故为O(nlnn)接着对于该节点需要更新邻接顶点的distance值，此时直接将那些更新了的push进堆即可(会和原来出现重复，但没关系)，这里我们需要维护小顶堆，对于n-1次循环我们共对每一条边都进行了插入,每次插入lnn,,故为O(mlnn).综上为O(nlnn)+O(mlnn)=O(mlnn)。当图为稀疏图时，这种优化是合算的，但当图是稠密图时，这时m=$O(n^2)$，整个复杂度变为$O(n^2lnn)$，比不优化还大，不合算。
      • Kruskal 算法：$O(mlogm)$，要求是稀疏图
        • 实现：需要优先队列和并查集。首先对权值排序，然后依次将最小权值的边加入到树中，保证加入的边不会在树中产生环，直到插入了n-1条边，这时就得到一棵生成树；遍历完所有的边，会得到一个最小生成森林
        • 是否存在环的判别：
          • 判断无向图是否有环：①拓扑排序（度为1入队） ②深搜（把顶点分为黑白灰三种颜色，如果遍历的过程中发现某个节点有一条边指向颜色为灰的节点，那么存在环。此方法可以把所有的环找出来）③并查集(遍历边，如果边的两个顶点的父亲一样，则存在环)
          • 判断有向图是否有环：①拓扑排序（入度为0入队） ②DFS
      • Boruvka’s Algorithm

    • 最短路径

      • BFS：（只适用于边权为正且相等的图），复杂度O(|E|) 。首先取一个点s入队，定义该点的距离为0，然后判断如果队列不为空，则队首出队，对于队首顶点的邻接顶点，如果其距离未被定义，则定义为队首顶点距离+1，并入队，循环直至队列为空。最后每一个点的距离就是从s到该点的最短距离。若存在顶点的距离未定义，说明从s到该点不可达，也就是图不连通。对于有向图，这种方法可以确定图是否强连通！

      • 拓扑排序：适用于 DAG

      • Dijkstra：适用于边权均为正数的图（有向无向均可），复杂度$O(n^2)$

        • 需要一个距离矩阵，除了source顶点初始化为0外都初始化无穷
        • 每一次我们访问一个顶点，都检查其邻接顶点，并记录最短路径，因此还需要一个visited数组和一个parent数组(注意在这里距离指的是和source点的最短距离，区别于最小生成树)
        • 每一次访问距离最小的顶点，标记其为visited，检查其所有unvisited的邻接顶点，判断是否当前顶点的距离加上边权小于该邻接顶点距离，若是更新distance和parent。循环n次，直至所有顶点都访问过，或者剩下的顶点的距离均是无穷
        • 复杂度：由于进行n次循环，每次循环需要找到当前距离最小的顶点，故为$O(n^2)$，而每次循环里面还必须对邻接顶点进行访问，共会有$O(m)$，故总的算法复杂度为$O(n^2)$

      • Bellman-Ford

      • SPFA(Shortest Path Faster Algorithm)：使用队列优化的Bellman-Ford算法

      • Floyd-Warshall：复杂度$O(n^3)$,“多源最短路径”问题
